# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in FinPro AI, please email the maintainers directly instead of using the public issue tracker.

Please include the following details:

- Description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Suggested fix (if available)

## Security Best Practices

When using FinPro AI:

1. **Protect Your API Keys**
   - Never commit `.env` files to version control
   - Rotate API keys regularly
   - Use environment variables for sensitive data

2. **Keep Dependencies Updated**
   - Regularly update Python and JavaScript dependencies
   - Monitor for security advisories

3. **Deploy Securely**
   - Use HTTPS in production
   - Implement proper authentication
   - Follow cloud provider security best practices

## Supported Versions

Security updates are provided for the latest version of FinPro AI.
